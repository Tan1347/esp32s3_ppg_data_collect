# ESP32-S3 内存占用情况分析

## 硬件资源

| 项目 | 值 |
|------|-----|
| 总 SRAM | 512KB |
| PSRAM | 8MB Octal |
| 应用可用 | ~400KB SRAM + 8MB PSRAM |

## 内存分布图

```
ESP32-S3 SRAM (400KB 可用) + PSRAM (8MB)
├── 系统开销 (~43KB SRAM)
│   ├── FreeRTOS Kernel: 8KB
│   ├── ESP System: 15KB
│   ├── NVS: 4KB
│   └── Flash Mapping: 16KB
│
├── 任务栈 (~55KB SRAM)
│   ├── main: 32KB ← 最大占用
│   ├── 常驻任务: 12KB (sys_led+ppg_led+button1+ble_cmd+logger)
│   └── 按需任务: 8KB (ppg+power+dht11)
│
├── BLE 协议栈 (~40KB SRAM)
│   ├── NimBLE Host: 25KB
│   ├── NimBLE Controller: 10KB
│   └── GATT/Queue: 5KB
│
├── 日志系统 (~8KB SRAM)
│   └── Ring Buffer + Sync: 8KB
│
├── SD 卡存储 (~60KB, 大部分在PSRAM)
│   ├── Write Buffers: 56KB (PSRAM)
│   └── SPI DMA + Sync: 4KB
│
├── UART 录制 (~48KB, 大部分在PSRAM)
│   ├── Buffers: 48KB (PSRAM)
│
├── WiFi 协议栈 (~74KB SRAM) ← 仅WiFi模式
│   ├── WiFi Driver: 30KB
│   ├── lwIP: 15KB
│   ├── Buffers: 24KB
│   └── HTTP Server: 5KB
│
└── 剩余可用
    ├── 无WiFi无SD: ~260KB SRAM + 8MB PSRAM
    ├── 有WiFi无SD: ~186KB SRAM + 8MB PSRAM
    └── 有WiFi有SD: ~126KB SRAM + 8MB PSRAM
```

## 实际测量值（从日志）

| 状态 | free_heap | free_psram | 说明 |
|------|-----------|------------|------|
| 启动后 | ~83KB | ~8MB | 常驻任务创建后 |
| BLE连接后 | ~21KB | ~8MB | NimBLE占用 |
| WiFi模式 | ~5KB | ~8MB | WiFi初始化后 |

## 问题分析

**为什么 free_heap 只有 21KB？**

理论计算 400KB - 43KB(系统) - 55KB(任务) - 40KB(BLE) - 8KB(日志) = **254KB 应该剩余**

实际只有 21KB，说明有 **~230KB 未被追踪的占用**。可能原因：

1. **main 任务栈 32KB** 是从 heap 分配的
2. **NimBLE 内部缓冲区** 可能比估计更大
3. **FreeRTOS 内部开销**（TCB、就绪队列等）
4. **Flash 映射到 DRAM** 占用额外空间（`.dram0.dummy` 段可达 92KB）
5. **编译器/BSS/DATA 段** 静态分配

**注意**: S3 有 8MB PSRAM，大缓冲区（>4KB）自动分配到 PSRAM，不占用宝贵的 SRAM。
