# ESP32-S3 固件架构分析

## 整体评价

项目采用标准 ESP-IDF 组件模型，13 个自定义组件由 `main.c` 作为状态机协调器。依赖图无循环，组件职责清晰。

**评分：9/10** — 架构合理，通信层已通过回调接口解耦，存储层类型依赖已消除。

---

## 架构优势

| 优势 | 说明 |
|------|------|
| 组件解耦 | 标准 ESP-IDF 组件模型，无循环依赖 |
| 回调架构 | ble_svc 和 wifi_transfer 通过 ble_callbacks_t / http_callbacks_t 回调接口通信，通信层零直接兄弟依赖 |
| 懒加载 | BLE/WiFi 按需初始化，节省资源 |
| 事件驱动 BLE | FreeRTOS 队列解耦 NimBLE 回调与业务逻辑 |
| 双缓冲存储 | SD 卡双缓冲 + LZ4 压缩 |
| 异步日志 | 环形缓冲区不阻塞传感器任务 |
| OTA 回滚 | 双分区 A/B，SHA-256 校验，失败自动回滚 |
| 指数退避 | WiFi 重连 1s→30s，最多 10 次 |

---

## 组件清单（13 个）

| 组件 | 职责 |
|------|------|
| main | 状态机协调器，注册所有回调，管理生命周期 |
| ppg_config | 全局配置、宏定义、类型定义（叶节点，被所有组件依赖） |
| ppg_algo | PPG 信号算法（心率/血氧） |
| max30102 | MAX30102 传感器 I2C 驱动 |
| sd_storage | TF 卡存储，双缓冲 + LZ4 压缩写入 |
| compress | LZ4 压缩库（仅被 sd_storage 使用） |
| ble_svc | BLE GATT 服务，命令收发 |
| wifi_transfer | WiFi HTTP 服务器，文件下载/OTA/状态查询 |
| wifi_prov | WiFi 凭据管理（NVS 持久化） |
| ota_upgrade | OTA 固件升级（SHA-256 校验 + 回滚） |
| ppg_log | 异步日志（环形缓冲 + 文件轮转） |
| battery | 电池电压 ADC 采集 |
| power_mgmt | 电源管理（DFS 动态调频） |
| uart_recorder | UART 数据录制（双缓冲 DMA） |

## 组件依赖图

```
main.c (协调器 — 注册所有回调，include 12 个组件头文件)
│
├── ble_svc ────────────→ ppg_algo (编译依赖: ppg_algo_result_t 类型)
│   └── 回调 ───────────→ battery, wifi_prov, sd_storage, ppg_log, uart_recorder
│
├── wifi_transfer ──────→ (零编译依赖，全部通过回调)
│   └── 回调 ───────────→ sd_storage, battery, wifi_prov, ota_upgrade
│
├── sd_storage ─────────→ ppg_algo (ppg_algo_result_t), compress (LZ4)
│
├── ota_upgrade ────────→ ppg_log (日志输出)
│
├── ppg_log ────────────→ (无兄弟依赖，直接 fopen/fwrite TF 卡)
│
├── wifi_prov ──────────→ (无兄弟依赖)
├── max30102 ───────────→ (无兄弟依赖)
├── battery ────────────→ (无兄弟依赖)
├── power_mgmt ─────────→ (无兄弟依赖)
├── uart_recorder ──────→ (无兄弟依赖)
├── compress ───────────→ (无兄弟依赖，LZ4 纯算法)
└── ppg_config ─────────→ (叶节点，所有组件依赖)
```
